# 前端管理

ai-fixer 提供 React 管理后台，用于查看系统状态、管理配置、查看 Incident 详情等。

## 页面导航

| 页面 | 路径 | 说明 |
|------|------|------|
| 仪表盘 | `/` | 系统状态概览 |
| Incidents | `/incidents` | 告警列表和详情 |
| 配置管理 | `/config` | 运行时配置 |
| 环境上下文 | `/environment` | 生产环境信息 |
| 插件 | `/plugins` | 插件管理 |

## 仪表盘

显示系统整体状态：

- **活跃 Incidents**：当前正在处理的告警数量
- **总 Incidents**：历史告警总数
- **插件数量**：已加载的插件数
- **LLM 信息**：当前使用的 LLM 提供商和模型
- **健康状态**：数据库、Redis 连接状态
- **SSE 连接**：实时事件流连接状态
- **最近 Incidents**：最新的 10 条告警

## Incidents 列表

查看所有告警及其处理状态。

### 筛选

- **状态**：new / diagnosing / awaiting_approval / executing / resolved / escalated
- **严重程度**：p0 / p1 / p2 / p3

### 详情

点击「详情」查看完整信息：

- **基本信息**：ID、状态、分类、严重程度、服务
- **诊断结果**：LLM 的根因分析、置信度
- **修复建议**：插件名、风险等级、描述
- **执行结果**：修复操作的执行状态
- **LLM 对话详情**：每一轮 LLM 的输入输出

### LLM 对话详情

按阶段分组显示：

- **Triage**：分类阶段的 LLM 交互
- **Diagnose**：诊断阶段的 LLM 交互（含工具调用）
- **Propose**：修复建议阶段的 LLM 交互

每个对话轮次显示：

- 角色标识（user / assistant / tool）
- 工具名称（如果是工具调用）
- 完整内容（Markdown 渲染）
- 工具输入（可展开）

## 配置管理

分组显示所有可配置项：

- **LLM 模型**：provider、api_key、model 等
- **飞书集成**：app_id、app_secret、alert_bot_ids
- **诊断配置**：置信度阈值
- **安全围栏**：命名空间白名单、配额限制
- **向量记忆**：embedding 配置
- **监控开关**：PostgreSQL、Redis、AWS
- **其他**：日志级别

### 修改配置

1. 修改表单中的值
2. 点击「保存修改」
3. 配置立即生效，无需重启

密钥字段显示为 `****`，修改时输入新值即可。

## 环境上下文

填写生产环境信息，帮助 LLM 更准确地诊断。

支持 Markdown 格式，建议包含：

- 服务列表及依赖关系
- 基础设施信息
- 告警严重程度定义
- 常见问题处理方式
- Shell 命令参考

## 插件管理

### 插件列表

显示所有已注册插件：

- **启用开关**：在线启用/禁用
- **名称**：插件唯一标识
- **描述**：功能说明
- **类型**：diagnostic / remediation
- **来源**：内置 / 自定义
- **风险等级**：low / medium / high / critical

### 操作

- **上传插件**：上传 `.py` 文件到自定义插件目录
- **热重载**：重新扫描插件目录
- **删除**：删除自定义插件（内置插件不可删除）
